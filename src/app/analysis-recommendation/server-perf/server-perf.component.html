    Backup Server

    According to the daemon configuration files provided by HSBC, servers marked with (*) in the spreadsheet are used mainly to host backup instances of components. These servers are not included in the server load analysis. HSBC should ensure that if there are failures, the servers dedicated to the backup instances of ION components have sufficient spare capacity to execute the processes.

    Hyper-threading

    In the preceding table, the servers highlighted in amber have HT enabled. HT increases the number of logical cores available on a server. However, all the tests conducted on ION software showed that HT does not provide any performance benefit and in fact reduces the accuracy of the reported CPU load statistics of servers and components. HT also increases the computational jitter. Therefore, it is not recommended for low-latency configurations.

    Power-saving configuration

    HSBC should verify whether any type of power-saving option (both at OS and BIOS level) is enabled on the servers used to host ION components. Recent tests performed by ION show that ION components perform better when power-saving features are disabled on the servers.

    Therefore, ION recommends that HSBC verify the following configurations on all the servers:

  • At the OS level, both C-State and P-State features should be disabled.
  • At the BIOS level, the power management should be set to the maximum performance mode.
